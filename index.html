Right-click any tile NEW

Mark as done parks a session you're not working with right now. The tile sinks to the bottom of auto-sort, its dot goes quiet, and — critically — background Stop / Notification / PreToolUse events from that Claude won't re-surface it or trigger passive-aggressive judgement. It stays parked until you submit a new prompt in that terminal; the next UserPromptSubmit is the only thing that un-parks it.

Clone terminal spawns a fresh copy of the same profile (cwd, shell, args). Set color picks a per-terminal theme override — click a swatch, no file editing needed. Kill terminal disposes it entirely.

Drag & drop to reorder NEW

Grab any tile, drop it where you want. A blue rail shows the insertion point. The order is saved to .claudelike-bar.jsonc and survives window reloads.

Sort modes NEW

Explicit sortMode switch. Auto floats "needs input" to the top. Manual respects your drag order. Dragging automatically flips the mode.

"auto"
"manual"

Auto-sort ranks by status: waiting → ignored → ready → working → done → idle. Manual uses per-terminal order values.

Auto-start terminals NEW

Mark projects with "autoStart": true and they spin up automatically on window open. Each terminal can override the global claudeCommand with its own.

Pair with VS Code's enablePersistentSessions and your Claude processes survive window reloads — no context loss.

Debug log NEW

Flip "debug": true in the config. The extension streams hook events, state transitions, and auto-start decisions to a Claudelike Bar output channel. The hook script also writes /tmp/claude-dashboard/debug.log. Useful when a tile seems stuck.

Configure it by talking to Claude

The config file is .claudelike-bar.jsonc in your workspace root — JSON with comments, auto-created when you first open a terminal. You can edit it by hand, but the easier path is to just tell Claude Code what you want. After installing, open a Claude terminal and say: "Walk me through configuring the Claudelike Bar."

You
Switch to passive-aggressive mode and rename api to "backend".
Claude
Done. Two edits to .claudelike-bar.jsonc:
"mode": "chill", "mode": "passive-aggressive", "terminals": { "api": { "color": "yellow", "nickname": null, "nickname": "backend", "autoStart": true } }
The extension picked them up live — no reload needed.
You
Auto-start my daily drivers: 3d-printing, api, life-planner, obsidian-vault. Run claude --dangerously-skip-permissions in each.
Claude
Four "autoStart": true flags set, and the global claudeCommand:
"claudeCommand": "claude --dangerously-skip-permissions", "terminals": { "3d-printing": { "autoStart": true, ... }, "api": { "autoStart": true, ... }, "life-planner": { "autoStart": true, ... }, "obsidian-vault": { "autoStart": true, ... } }
Next time you open VS Code, all four launch and run Claude automatically.
You
Vault Direct needs a different startup command — it's my Telegram bridge.
Claude
Added a per-terminal override. This one inherits nothing from claudeCommand:
"Vault Direct": { "color": "blue", "autoStart": true, "command": "claude --channels plugin:telegram@claude-plugins-official" }
You
Something's weird — my life-planner tile is stuck on "Working" forever. Can you figure out what's wrong?
Claude
Turning on debug logging:
"debug": false, "debug": true,
Open the Claudelike Bar output channel (Ctrl+Shift+U) and run a prompt in life-planner. I'll watch for hook events and state transitions — if the hook never fires or the project name doesn't match the tile, we'll see it in the trace.

Everything you see in these conversations is plain JSONC in one file. Comments, section headers, and defaults are preserved by the extension's round-trip save — feel free to hand-edit. But most days it's easier to just ask.
